# TD Organisation du projet

Maintenant vous allez créer et organiser le dépôt.

1. [x] Creer un projet dans gitlab avec la règle : GL2020-<IOTIA|DAM>-<'Nom du groupe sous moodle'> 
2. [x] Récuperer dans le gitlab du cours le [template pour les user stories](https://git-iutinfo.unice.fr/blay/quizzfordisabledpeople/blob/master/.gitlab/issue_templates/UserStory.md).
4. [x] **Ecriture des histoires utilisateurs** sous la forme d'issue dans gitlab en respectant le template
      * [x] Certaines histoires peuvent rester sous la forme d'EPIC, dans ce cas vous ne les traitera pas dans le 1e sprint
      * [x] Vous devez travailler sur un ensemble d'histoires telles que : 
          * Vous puissiez construire un MVP (Produit minimum viable) dès le Sprint 1;
          * Chaque étudiant ait la responsabilité d'au moins une histoire à traiter.
4. [x] Structurer le dépôt de code de façon à bien séparer les documents des codes
5. [x] **Prévoir les milestones conformément au moodle**
6. [x] Prévoir les livrables attendus (devoirs) avec en particulier les dates de rendus
7. [x] Répartir les US dans les milestones au moins dans les premiers sprints
8. [x] Evaluation de la complexité des US et affectation des responsabilités si ce n'est pas fait
9. [ ] Individuellement : **Séparation des US en tâches de développement** sur papier au moins
10. [ ] En groupe : Discussions sur les tâches de développement 
      * [ ] Saisie des tâches de développement sous la forme d'issues (Ne pas hésiter à forcer l'utilisation d'un template dédié)
      * [ ] Association des tâches de développement aux histoires utilisateurs
      * [ ] Définition si besoin de nouveaux labels (e.g. Bug, Documentation, Livrable, ....)
11. [ ] Mise en place d'un Kanban : choix des catégories (e.g., ToDo, ToTest, Bug, Done, Closed, etc.)
9. [ ] Rapport : documentation des tâches : doc utilisateur; doc techniques
10. [ ] Poser un prototype (un code minimum du type Hello mais adapté au projet) assurant que vous avez bien mis en place les tests et l'intégration continue.
